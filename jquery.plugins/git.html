Submodules

添加一个submodule
git submodule add git://github.com/rack/rack.git rack
.gitmodules文件
记录着项目中所有submodules的信息。

一个submodule其实只是一个指向该submodule的源的某一个commit的指针。拥有该submodule的git项目并不会去跟踪其内容。submodules和父项目之间是被分开看待的两个项目，git命令在两者之间运行会得到不一样的结果。必须紧记这点。

克隆有submodules的项目。

项目克隆下来后，里面的submodules只有目录，并没有内容。必须执行以下两步：

  1. git submodule init
  2. git submodule update

当submodules有了更新之后，执行以下两步：

  1. git merge origin/master
  2. git submodule update

当有人修改了submodules但不提交到public repository的话，执行git submodule update就会发生错误。
